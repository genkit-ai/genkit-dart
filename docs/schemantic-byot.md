# Bring Your Own Types (BYOT) in Genkit Dart

Currently, Genkit Dart relies on the `schemantic` package for defining input/output schemas for flows, tools, and prompts. While `schemantic` provides a powerful way to define schemas that generate both Dart types and JSON Schemas, some users prefer using other popular serialization libraries like `json_serializable`.

To support this, we've relaxed the `SchemanticType` restriction and provided a helper to bridge any JSON Schema and parser into Genkit.

## Using `json_serializable` (6.13.0+)

The latest version of `json_serializable` supports built-in JSON Schema generation. This is the most ergonomic way to use 3rd-party types with Genkit.

### 1. Define your class

Annotate your class with `createJsonSchema: true`:

```dart
import 'package:json_annotation/json_annotation.dart';
import 'package:schemantic/schemantic.dart';

part 'person.g.dart';

@JsonSerializable(createJsonSchema: true)
class Person {
  final String firstName;
  final String lastName;

  Person({required this.firstName, required this.lastName});

  factory Person.fromJson(Map<String, dynamic> json) => _$PersonFromJson(json);
  Map<String, dynamic> toJson() => _$PersonToJson(this);

  /// The JSON Schema generated by json_serializable.
  static const jsonSchema = _$PersonJsonSchema;

  /// Bridge into Genkit/Schemantic
  static final schema = SchemanticType.from<Person>(
    jsonSchema: Person.jsonSchema,
    parse: (json) => Person.fromJson(json as Map<String, dynamic>),
  );
}
```

### 2. Use it in Genkit

```dart
final ai = Genkit();

ai.defineFlow(
  name: 'greet',
  inputSchema: Person.schema,
  outputSchema: .string(),
  fn: (person, _) async => 'Hello, ${person.firstName}!',
);
```

## Manual Schema Definition

If you are using an older version or another library that does NOT generate JSON Schema, you can provide the schema manually using `SchemanticType.from`:

```dart
final manualSchema = SchemanticType.from<MyType>(
  jsonSchema: {
    'type': 'object',
    'properties': {
      'foo': {'type': 'string'},
    },
    'required': ['foo'],
  },
  parse: (json) => MyType.fromJson(json as Map<String, dynamic>),
);
```

## Why this change?

By relaxing `SchemanticType` from a `base` modifier, we allow non-generative implementations. This decouples Genkit's core from the specific `schemantic` code-generator while still maintaining full type safety and JSON Schema compatibility for the Genkit UI and protocol.
